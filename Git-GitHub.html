Why use versioncontrol with git and GitHub?

  • Safety and Control
  • Collaboration
  • Community

Safety and Control

A typical project might look something like this.
What is lacking:
  • What changed? files must be manually compared
  • When was the change made? is implicitly captured by the system, but is not reliable
  • Who made the change? very important when collaborating
if we switch to Git for version control:
  • Changes are captured with commits
  • The filenames remain unchanged
  • Commits are identifed uniquely with automatically generated sha1 hashes, which are guaranteed to be unique
  • these are usually abbreviated to the first 6 characters
Each commit captures the following information automatically
  • Who made the commit?
  • When was the commit made?
  • What changed precisely ?
  • Optional: why?
Each commit requires the user to input information on why the change was made

Collaboration

  • Git makes collaboration smoother
  • Git repositories can easily be synced with Git servers (e.g. GitHub, Bitbucket or Gitlab)
  • Track changes works online as well as offline
  • Conflicts are rare 1 and very specific 2
  • 1 Conflicts only occur when the same line has been changed.
  • 2 Conflicts are specific to that line
  • Note: Github != Git
